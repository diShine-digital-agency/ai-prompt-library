# Client communication templates

Prompts for drafting professional client communications — especially the ones
nobody likes writing. Scope creep pushback, fee increases, project delays,
and diplomatic disagreements.

## When to use

- you need to deliver bad news without damaging the relationship
- scope creep is happening and you need to address it
- it's time for a fee increase conversation
- a project is delayed and the client needs to know
- you're inheriting a messy client relationship and need to reset expectations

## Template: scope creep pushback

```
Draft a professional email to a client who has been requesting work
outside our agreed scope.

**Context:**
- client name: {{client_name}}
- project: {{project_description}}
- original scope: {{original_scope}}
- what they've been requesting: {{extra_requests}}
- relationship temperature: {{relationship}} (good/neutral/tense)
- my communication style: {{style}} (direct/diplomatic/formal)

The email should:
1. acknowledge their requests positively (don't make them feel wrong for asking)
2. reference the original scope without sounding legalistic
3. clearly explain what falls outside scope
4. offer a constructive path forward (change order, separate engagement, next phase)
5. include a specific next step (call, proposal, etc.)
6. maintain warmth — I want to keep this client

Tone: firm but collaborative. Never passive-aggressive.
Length: under 200 words. Clients don't read long emails.
```

## Template: fee increase communication

```
Draft a message to inform {{client_name}} about a fee increase.

**Details:**
- current rate/fees: {{current_fees}}
- new rate/fees: {{new_fees}}
- increase percentage: {{percentage}}
- reason: {{reason}} (e.g., "annual adjustment", "expanded scope", "market alignment")
- effective date: {{effective_date}}
- advance notice period: {{notice_period}}
- relationship length: {{years}} years
- any sweetener I can offer: {{sweetener}} (e.g., "locked for 12 months", "added deliverable")

The message should:
1. lead with value delivered (remind them what they're getting)
2. present the increase matter-of-factly (not apologetically)
3. explain the why briefly — one sentence, not a justification essay
4. include the sweetener naturally
5. give them time to adjust (reference the notice period)
6. end with continuity language ("looking forward to continued work together")

Do NOT: apologize excessively, list market data to justify it,
or use the word "unfortunately."
```

## Template: project delay notification

```
Draft a message to {{client_name}} about a project delay.

**Situation:**
- what's delayed: {{deliverable}}
- original deadline: {{original_date}}
- new expected date: {{new_date}}
- reason: {{reason}}
- is this our fault or external? {{fault}}
- impact on other deliverables: {{downstream_impact}}
- what we're doing to minimize impact: {{mitigation}}

The message should:
1. state the delay clearly in the first sentence (don't bury it)
2. own it if it's our fault — no blame-shifting
3. explain the reason briefly (one sentence)
4. present the new timeline with confidence (don't hedge with "hopefully")
5. show what we're doing to prevent future delays
6. offer a call if they want to discuss
7. if possible, deliver a partial result or progress update alongside the bad news

Never: blame specific team members, give technical excuses the client
doesn't care about, or stack multiple pieces of bad news.
```

## Template: difficult conversation preparation

```
I have a difficult conversation coming up with {{client_name}}.

**The situation:**
{{situation}}

**What I need to communicate:**
{{my_message}}

**What I think they'll push back on:**
{{expected_pushback}}

**What I absolutely need from this conversation:**
{{must_haves}}

**What I'm willing to be flexible on:**
{{negotiable}}

Help me prepare:
1. an opening that sets the right tone (collaborative, not confrontational)
2. my key points in order — lead with the least contentious
3. prepared responses for their likely objections (don't argue, redirect)
4. the specific language to use for the hardest part
5. phrases to avoid (trigger words that escalate)
6. a good closing that reaffirms the relationship
7. a follow-up email template to send after, summarizing what was agreed
```

## Template: project status update

```
Write a project status update email for {{client_name}}.

**Project:** {{project}}
**Reporting period:** {{period}}

**Completed this period:**
{{completed}}

**In progress:**
{{in_progress}}

**Blocked or at risk:**
{{blocked}}

**Key metrics (if applicable):**
{{metrics}}

**Next steps and upcoming milestones:**
{{next_steps}}

Format rules:
- keep it scannable — bullets, not paragraphs
- lead with progress, not problems
- for blocked items, always include what we need from them (if anything)
- include one "win" or positive insight they'll want to share with their boss
- end with clear next actions and owners
- total length: under 300 words
```

## Template: resetting expectations with a struggling project

```
A project with {{client_name}} is not going well and we need to reset.

**What went wrong:**
{{issues}}

**Current state:**
{{current_state}}

**What needs to change for success:**
{{changes_needed}}

**What I need from the client:**
{{client_needs_to_do}}

Draft a message or talking points for a reset conversation that:
1. acknowledges the situation honestly without dwelling on blame
2. shows I've diagnosed the root cause (not just symptoms)
3. presents a clear corrective plan with specific actions
4. identifies what needs to change on both sides
5. sets new milestones that are realistic (not optimistic)
6. rebuilds confidence through specificity, not reassurance
```

## Tips

- write the hardest sentence first, then build the rest of the message around it
- for fee increases: timing matters. Don't do it right after a problem. Do it after a win.
- for delays: always pair bad news with a concrete mitigation or partial delivery
- never send a difficult email on Friday afternoon — the client stews all weekend
- if a conversation goes badly, follow up within 24 hours with a written summary
  that reframes things constructively

## Common mistakes

- over-apologizing, which undermines your position and makes clients more nervous
- burying the lead — put the key information in the first two sentences
- explaining too much. Clients want to know the impact on them, not the technical details.
- sending a long email when a 10-minute call would be faster and less prone to misinterpretation
- using templates verbatim without personalizing for the specific relationship
